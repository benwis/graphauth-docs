Logs in user based on username or email, and password
Generates new access and refresh tokens for the user.
This can use take a password, or a magic link token if that feature is enabled.
One of the two must be provided.
If user approval is required, and the user is not approved, login will throw an error.
If you'd like to login with webauthn, see the webauthn login mutation.

```graphql
login(
  name: String
  email: String
  tenantName: String!
  password: String
  magicToken: String
): LoginPayload!
```
